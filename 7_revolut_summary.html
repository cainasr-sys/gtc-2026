07 — Revolut Financial Foundation Model
Revolut · NVIDIA GTC · Mar 2026

PRAGMA.

Pre-trained General Model for Autonomous Representation of Arbitrary customer behavior — o modelo fundacional de transações da Revolut

Apresentado por Revolut Data Science
Dataset 850B+ tokens
Usuários 25M+ / 20 países
Categoria Foundation Model
01 / O Problema

ML tradicional em finanças
é fundamentalmente quebrado

Entender o domínio

Meses de coleta de expertise específica por domínio — fraude, crédito, AML e marketing são silos separados com times separados.

Feature engineering manual

Centenas de features hand-crafted por modelo. Semanas ou meses para cada novo projeto. Nenhum reaproveitamento entre domínios.

Validar e manter pipelines

Statistical testing, drift monitoring, ETL jobs, feature stores e versionamento — infraestrutura pesada para cada modelo individualmente.

Zero transferência de conhecimento

O que o modelo de fraude aprende não ajuda o modelo de crédito. Inconsistência em definições de features semanticamente idênticas.

Exemplos de features típicas construídas manualmente:

count_transactions_last_7d ratio_international_txn std_amount_by_hour_of_day avg_amount_merchant_category_30d time_since_last_txn_seconds unique_merchants_90d
02 / A Visão

Transações são
como linguagem

Propriedade Linguagem natural Transações financeiras
Sequential Palavras em sentenças Transações ordenadas no tempo
Contextual Significado vem das palavras vizinhas Comportamento emerge das transações vizinhas
Patterns Gramática, idiomas, estilo Ritmos de gasto, hábitos de consumo
Anomalies Erros gramaticais, typos Atividade fraudulenta ou incomum
Long-range deps Coerência no nível do documento Padrões sazonais, life events
Se o masked language modeling (BERT) consegue aprender representações universais de linguagem, masked transaction modeling pode aprender representações universais de comportamento financeiro — sem nenhum label supervisionado.

Path 1 — Embeddings como Features

Sequence → Frozen PRAGMA → Embeddings → Boosting probe → Prediction

PRAGMA congelado. Qualquer modelo downstream pode usar os embeddings como input. Deploy mais rápido — sem retreino do foundation model.

Path 2 — Fine-tuning com LoRA

Sequence → PRAGMA + LoRA adapters → Prediction

Apenas ~1-2% dos parâmetros treináveis. Um modelo base + N adapters LoRA pequenos = N modelos especializados, com baixo custo e sem catastrophic forgetting.

06 / Key Takeaways

O que levar
daqui

01

Transações são sequências

A analogia com NLP é profunda e válida — as mesmas propriedades estruturais que tornam o BERT poderoso para texto valem para histórico transacional.

02

Tokenização é crítica

Como você converte os dados brutos em tokens discretos determina o teto de performance do modelo. A escolha do BPE para texto via Nemotron foi um diferencial mensurável.

03

Pré-treinamento auto-supervisionado

Bilhões de transações sem qualquer label são suficientes para aprender representações comportamentais ricas — o modelo aprende o que importa sozinho.

04

Dois caminhos de deployment

Embeddings como features (rápido, flexível) ou fine-tuning com LoRA (mais performance, ainda eficiente). A escolha depende do caso de uso e exigência de latência.

05

Ganhos massivos de eficiência

O verdadeiro ROI não é apenas na performance dos modelos — é na eliminação de semanas/meses de feature engineering por cada novo projeto de ML.

06

Um modelo, muitas tarefas

Fraude, crédito, AML, cross-sell e comunicação se beneficiam das mesmas representações aprendidas. Padrões de fraude informam modelos de crédito, e vice-versa.
